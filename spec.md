# "人有三急"小游戏设计方案

## 1. 游戏概述

**游戏名称**：人有三急  
**平台**：Web / H5（GitHub Pages 部署）  
**屏幕适配**：自适应浏览器窗口尺寸，全屏 Canvas 渲染  
**美术风格**：像素风（程序化 Canvas 2D 绘制，无外部图片资源），整体色调柔和、干净、可爱，完全避免写实或低俗感。  
**核心体验**：每一次进入，都面对一个随机生成的小便池队列，玩家只需点击一个空位，完成"选择"，获得轻松幽默的反馈。无竞争、无积分，仅以日常幽默感和共鸣为乐趣。

---

## 2. 场景与视觉设计

### 2.1 场景构成

- **背景**：一面瓷砖墙壁（暖灰色 `#D4C5A9`），交错墙砖网格纹理；踢脚线；像素地板（棕色系），地板砖缝。
- **尿兜**：沿墙壁从左到右水平等距排布，每个尿兜通过 Canvas 程序化绘制（陶瓷主体、排水盆、排水孔、上沿高光、左右装饰线、入墙水管）。儿童尿兜（可选）尺寸为正常 60%，配浅蓝色调。
- **隔板**（可选）：尿兜之间的隔板，有**长隔板**（从尿兜顶部延伸到地板附近）和**短隔板**（从尿兜中部延伸到地板附近）两种模式，随机生成。
- **标语牌**（可选）：两种模式——① 每个尿兜上方独立小标语牌；② 墙中央统一大标语牌（"向前一小步，文明一大步"）。
- **场景亮度**（随机）：正常亮、半亮（傍晚/黎明色调）、昏暗（夜晚蓝色调）三种。
- **占位 NPC**：部分尿兜前站立像素小人，背对镜头，有**体型差异**（瘦/正常/壮）、**身高差异**（`heightScale: 0.85~1.12`）、**颜色组合**（10 种帽色×上衣色×发色 + 8 种裤色 + 4 种鞋色）。儿童尿兜前的 NPC 为**儿童体型**（`heightScale: 0.55~0.65`，始终瘦型，鲜艳配色）。
- **玩家角色**：橙色棒球帽 + 橙色上衣 + 蓝色牛仔裤，选择空位后从画面左边缘走入。

### 2.2 视觉情绪

- 暖灰色、浅蓝、白色为主色调，营造干净公共洗手间氛围。
- 人物为像素方块头，不同角色通过帽子、发色、上衣颜色简单区分。
- UI 极简：顶部提示文字 + 底部按钮（跳过/重来/下一泡），气泡文字 `sans-serif` 粗体。

## 3. 核心玩法与流程

### 3.1 单局流程

1. **进入游戏**：自动随机生成当前男厕场景（尿兜数量 3~8 个，随机亮度、隔板、标语、儿童尿兜）。
2. **观察场景**：玩家看到一排尿兜，其中部分已被 NPC 占据，部分空着。
3. **做出选择**：点击任意尿兜。
4. **反馈呈现**：
   - **点击空位**：玩家角色从左侧走入，缓动插值 + 行走起伏 + 手臂摆动动画，走到目标位置后站定，播放成功音效，弹出随机夸赞气泡（8 条之一："完美！""明智之选！""懂得都懂！""讲究！""标准答案！""稳！""真男人！""有品位！"）。
   - **点击有人位置**：该 NPC 转头看向玩家（头部偏移 + 表情绘制），播放被拒音效，弹出随机嫌弃气泡（5 条之一："？""…""?!""喂！""看啥呢"），约 1 秒后恢复。
5. **一轮结束**：成功选择后显示"下一泡"按钮，点击刷新场景。也可在 IDLE 时点击"忍一时风平浪静"按钮跳过当前场景。

### 3.2 隐含的趣味点

- 玩家内心会不自觉使用"小便池社交礼仪"（不挨人、优先角落等），随机布局天然制造纠结和幽默。
- 每次重来期待看到不同阵型，形成自发重玩。
- 被拒时 NPC 转头瞪眼的反应增加互动趣味。

## 4. 代码架构与模块说明

项目采用**单页面 Canvas 2D** 架构，无外部依赖，入口为 `index.html`。

### 文件结构

```
├── index.html                    // 入口页面
├── game.js                       // 导入 Main，启动游戏
├── js/
│   ├── main.js                   // 游戏主类（场景生成、渲染、交互、动画，约860行）
│   ├── databus.js                // 游戏状态管理器（单例）
│   ├── render.js                 // Canvas 创建 & 屏幕尺寸管理
│   └── runtime/
│       └── music.js              // 音效管理器（单例）
├── audio/
│   ├── bgm.mp3                   // 环境背景音（循环）
│   ├── bullet.mp3                // 成功音效
│   └── boom.mp3                  // 被拒音效
└── images/                       // （原微信小游戏遗留资源，代码中未使用）
```

### 模块一：场景数据生成（`main.js` → `initScene()`）

- 随机尿兜数量 `count：3~8`
- 随机场景特征：亮度（3 级）、隔板（3 类）、标语（3 模式）、儿童尿兜（3 位置）
- 随机占据状态：先随机总人数 `M = 0 ~ count-2`，再从所有尿兜中随机选 M 个设为"有人"（保证至少 2 个空位）
- 每个尿兜生成完整数据结构：

```js
{
  id,                    // 索引
  x, y, width, height,  // 尿兜矩形（命中检测 + 绘制）
  occupied,              // 是否有人
  style,                 // NPC 外观对象（null 表示空闲）
  cx,                    // 人物/尿兜水平中心
  personIdlePhase,       // 待机呼吸动画相位
  hasSign,               // 是否有小标语
  isChild,               // 是否为儿童尿兜
}
```

### 模块二：场景渲染（按绘制顺序）

1. **`drawBackground()`** — 墙壁 + 墙砖网格 + 踢脚线 + 地板 + 地板砖缝
2. **`drawPartitions()`** — 可选隔板（长/短），每个尿兜之间
3. **`drawBigSign()`** — 可选大标语牌（模式 2）
4. **`drawUrinals()`** — 所有尿兜（含儿童尿兜缩放 + 小标语牌）
5. **`drawNPCs()`** — 所有占据位置的 NPC（背向，带呼吸 bob + 转头动画）
6. **`drawPlayer()`** — 玩家角色（选择后从左侧缓动走入，行走起伏 + 摆臂）
7. **`drawBubble()`** — 对话气泡（含淡出动画 + 三角尾巴）
8. **场景亮度遮罩**
9. **`drawUI()`** — 顶部提示 + 底部按钮

### 模块三：交互与输入处理（`main.js` → `bindTouch()` + `handleTap()`）

- 绑定 `pointerdown` + `touchstart` 事件
- 点击检测区域：尿兜矩形 ±10px + 从尿兜顶部到人物脚底的整个垂直范围
- 状态机控制交互：
  - `IDLE` → 可点击尿兜，也可点击"忍一时风平浪静"跳过
  - `ANIMATING` → 禁止交互，等待播放完成
  - `DONE` → 可点击"下一泡"重来

### 模块四：动画与音效

**NPC 待机动画**：
- 呼吸上下浮动：正弦波 ±1.5px
- 微步重心偏移：正弦波 ±1.5px
- NPC 身高缩放（`heightScale`）+ 体型宽度因子（瘦: 0.78 / 正常: 1.0 / 壮: 1.28）

**成功动画**：
- 玩家缓动走到目标位置（二次缓动函数 `ease`）
- 行走起伏 ±2.5px + 手臂摆动 ±5px
- 到达后站立，弹出绿色夸赞成气泡（不消失）

**拒绝动画**：
- NPC 头部向右偏移最多 6px
- 眼睛和嘴部表情绘制（`turn > 0.4`）
- 弹出橙色被拒气泡，淡出消失（`timer < 12` 时 alpha 渐变）
- 50 帧后恢复

**音效**（`Music` 单例）：
- 环境背景音 `bgm.mp3`（循环，20% 音量）
- 成功音效 `bullet.mp3`
- 被拒音效 `boom.mp3`

### 模块五：状态管理（`DataBus` 单例）

```
IDLE ──点击空位──→ ANIMATING ──动画结束──→ DONE
  │                                              │
  │                                              │ 点击"下一泡"
  └──点击有人──→ REJECTING ──动画结束──→ IDLE     │
  └──点击"忍一时风平浪静"──→ 重新生成场景 ──→ IDLE  │
                                                 ↓
                                              IDLE（重新生成）
```

## 5. 关键技术实现

- **渲染引擎**：原生 Canvas 2D API，所有图形程序化绘制（`fillRect`、`strokeRect`、`beginPath`、`fillText`），零外部图片资源
- **屏幕适配**：Canvas 尺寸 = `window.innerWidth × window.innerHeight`，监听 `resize` 事件实时更新。布局基于 `SCREEN_HEIGHT` 比例计算（地板 Y=78% 高度、人物高度=28.5%等）
- **像素风格**：通过固定像素值的矩形和线条模拟像素纹理，无抗锯齿依赖
- **模块化**：ES Module 导入导出，`Main`、`DataBus`、`Music` 均为单例/类
- **性能**：单循环 `requestAnimationFrame`（update → render），所有绘制在每帧执行
- **无外部依赖**：纯原生 JavaScript，无需打包工具，可直接静态部署

## 7. 注意事项

- 保持趣味性但不冒犯：所有反馈为轻松、自嘲式的幽默，不包含任何羞辱或不适表达。
- 动画简洁，帧率不受限。
- "有人"角色仅作为场景元素存在，无性别、身份等深入刻画，避免争议。
- 游戏为静态网站，可直接通过 GitHub Pages 等平台部署，无需后端服务。
